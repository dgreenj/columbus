# Tickets Site Integration Design

**Date:** 2026-04-18  
**Project:** columbus (Italy Trip 2026 Quarto site)  
**Goal:** Integrate the Florence ticket agent output into the site as curated Tickets pages under Pre-Trip Planning, expandable to Naples.

---

## Overview

The ticket agent (`scripts/ticket_agent.py`) already scrapes official booking pages and writes `research/tickets-days-3-4.md`. The current output is noisy (social links, nav text, double bullets). This design fixes the agent to produce clean site-worthy markdown, splits output into per-city files, and wires two new `.qmd` pages into the site navigation.

---

## Part 1: Agent Improvements

Five targeted fixes to `scripts/ticket_agent.py`:

### 1.1 Price extraction — regex instead of keyword heuristic

**Current:** matches any line containing `€`, `euro`, `admission`, `price`, `ticket`, `fee`  
**Fix:** require `€\d` pattern (euro sign immediately followed by a digit) using `re.search(r'€\d', line)`  
**Effect:** eliminates nav text, link labels, and heading lines; only actual price amounts pass through

### 1.2 Booking link filter — domain whitelist

**Current:** matches any markdown link whose text/URL contains `book`, `buy`, `ticket`, `reserv`, `shop`, `acquist`  
**Fix:** extract the URL from the markdown link and check it against a whitelist of known ticket domains:
- `tickets.duomo.firenze.it`
- `uffizi.it`
- `galleriaaccademiafirenze.it`
- `b-ticket.com`
- `museumflorence.com`
- `operamedicealaurenziana.org`
- `coopculture.it`
- `ticketone.it`
- `vivaticket.com`

**Effect:** social media, Facebook share links, and navigation links are excluded entirely

### 1.3 Fix b-ticket.com redirect URLs

**Current:** Accademia, Medici Chapels, and San Marco all point to `https://www.b-ticket.com/b-ticket/uffizi/default.aspx` (the generic portal root, which redirects)  
**Fix:** replace with direct booking URLs — implementer should use firecrawl to verify the correct current URL for each during implementation:
- Accademia: use `galleriaaccademiafirenze.it` official site
- Medici Chapels: use Polo Museale della Toscana booking page
- San Marco: use Polo Museale della Toscana booking page

### 1.4 Double-bullet fix

**Current:** scraped markdown list items begin with `- ` or `* `; `write_report()` prepends another `- `, producing `- - [Link](url)`  
**Fix:** in `extract_ticket_info()`, strip leading `- `, `* `, and `• ` from each line before storing  
**Effect:** report renders as clean single-level bullet list

### 1.5 Split output by city

**Current:** single output file `research/tickets-days-3-4.md`  
**Fix:**
- Add `"city"` field (`"florence"` or `"naples"`) to each entry in `SITES`
- `write_report()` accepts a city filter and writes to a city-specific path
- `__main__` calls `write_report()` twice: once for florence → `research/tickets-florence.md`, once for naples → `research/tickets-naples.md`
- Add Naples venues to `SITES` (derived from Days 8–9 itinerary files: Herculaneum/MANN/Pompeii/Castel Nuovo/Cappella Sansevero)

## Part 2: Site Pages

### `research/tickets-florence.qmd`

```yaml
---
title: "Florence Tickets & Booking"
subtitle: "Days 3–6 · June 7–10"
categories: [planning, florence, tickets]
toc: true
---
```

Body:
1. Brief intro (1–2 sentences: book 2–3 months ahead, June weekends sell out fast)
2. Quarto tip callout box:
   ```
   **To refresh this data:** run `python3 scripts/ticket_agent.py` from the project root, then re-render the site.
   ```
3. `{{< include tickets-florence.md >}}`

### `research/tickets-naples.qmd`

```yaml
---
title: "Naples Tickets & Booking"
subtitle: "Days 8–9 · June 12–13"
categories: [planning, naples, tickets]
toc: true
---
```

Same structure: intro + callout + `{{< include tickets-naples.md >}}`

## Rendering

Site renders via:
```bash
docker exec rstudio quarto render /home/rstudio/Quarto/columbus
```

The `.qmd` pages use `{{< include >}}` (a Quarto shortcode), not R/Python execution — no container dependencies on firecrawl. The agent runs on the host; the site render just includes the pre-generated `.md` files.

---

## Out of Scope

- Puglia tickets (tour group handles bookings)
- Live/interactive ticket availability
- Automatic agent re-run on render
